# OpenSpec authoring rules

- Keep normative requirements in `spec.md` using RFC-style MUST/SHALL language.
- Use explicit `#### Scenario:` sections with WHEN/THEN outcomes.
- Keep one active change unless explicitly approved; do not pre-create roadmap
  change directories.
- Keep tasks ordered, test-first, small, and end with review/fix, verification,
  and documentation reconciliation.
- Never fabricate evidence in `verify.md`; record pending status until checks run.
